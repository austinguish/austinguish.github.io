也谈贪心证明（其实是翻译Stanford Lecture Notes

贪心的哲学

「目光短浅」「运气好」「但是在每一个阶段贪心得到的结果都不差于其他算法」

可以用贪心的特征

在一定的「约束」下，通过一些序列化的操作可以获得一个满足条件的最值。

  • 输出一个你能参加的活动的最大数量，在你不能参加overlapping 活动的前提下
  • 使得你所选的边的权值和最小，在选择的边属于Spanning Tree 的前提下

贪心证明的两个要素

  • Feasible 也就是你得说明你每一步采取的行动得是满足一定约束的（比如满足一定的顺序，满足不会出现overlap….）
  • Optimal 得证明这是不差于最优的，得到了一个「最值」

证明optimal 的方法

Greedy Stays Ahead

通过一些 measure, 表明贪心算法在每一次迭代中至少不差于最优解。大部份都是用归纳+反证。

  1. 定义你的解。你的解产生了一系列 对象 $X$, 与最优解产生的 $X^*$. 引入一些变量
  2. 定义你的measure $S = {m_1(X),m_2(X)….m_n(X)}$ $S^* = {m_1(X^*),m_2(X^*)…m_k(X^*)}$
  3. 证明 你的measure 在每一步$m_i(X) $ 不次于 $m_i(X^*)$ 通过归纳法 （一般）
  4. 证明最优 用反证法 假设贪心得到的不是最优 ，但是你的贪心比最优解更好，所以你的就是最优

例题1：

​ 在一维公路的 $𝑥_1 < ⋯ < 𝑥_𝑛 $处各有一栋房子, 现在要在公路上设置基站来覆盖所有的房子. 若基站设置在 𝑡 处, 则能覆盖 [𝑡 − ℎ, 𝑡 + ℎ], 其中 ℎ > 0 为给定常数. 设计算法给出一个设置方案, 使得需要的基站数目最小。

  • 策略：尽可能晚放基站，保证基站前h范围内的房子都能覆盖。下一个放置位置是从上一个基站不能覆盖的第一个房子的位置$+h$ 开始。也就是说在相邻两个基站之间，下一个基站是满足这两个基站之间所有房子被覆盖的最远距离。（不重叠或者说是让两个基站不重复服务）

  • Measure（同样基站数量下,贪心覆盖的房子更多。或者同样覆盖的房子下，贪心用的基站更少）这里证明第一种

  • Lemma 设对于前$i$ 个基站， 贪心设置的基站位置为 ${s_1,s_2…..,s_j}$ 最优解设置的基站位置是${t_1,t_2…..,t_k}$, $\forall 1 \le i \le min{j,k}$, 我们有$s_i\geq t_i$。（下标代表基站数量，所以对于相同的覆盖程度 下标越小越好）

  • proof 归纳法

    当 $i = 1$ 时，$ j = k = 1$ 都放一个基站就好了。

    假设Lemma 在 $i = n$ 的时候成立，我们有$s_n\ge t_n$, 如果我们想要覆盖$[s_n,t_{n+1}]$之间的房子，$s_{n+1}$ 是区间内最远的放置位置, 在这个区间内我们有$s_{n+1}\ge t_{n+1}$。这里的意思就是说抛开$s_n, t_n$左侧覆盖的房子。$s_n+h 到$ $t_n+h$ 这个小段. 为了能覆盖到，$t_{n+1} $肯定比$ s_{n+1}$ 提前,(上一个覆盖的少了)。就相当于你写暑假作业，之前使劲玩，后面才开始写，你就得提前抓紧。

  • Proof $j\le k$. 用反证法 假如 j>k ,那么s不能覆盖所有的房子，t 更不可能 因为$s_i\geq t_i$ 所以 $j\le k$

Exchage Arguments

goal 迭代的把最优解变成贪心解，同时最优解的cost不变。

交换论证最基本情况聚焦于相邻逆序对的交换, 这种方 法在运用的时候一般比较机械, 计算就可以了.

套路

  1. 清楚定义贪心解和最优解的变量参数。
  2. 说清楚两种解的区别。比如 一个有，另一个没有。 一个与另一个的顺序不同
  3. 交换一部分（交换不同的地方），同时没有增加或者改变Cost
  4. 迭代 证明经过很多次迭代它变成了最优解。
